# Implementation Guide

The implementation guide is the central artifact of the strategic-implementation workflow. It is created by the orchestrator after architecture and UX/PMF review, refined by the full agent panel, and approved by the user before any session plan is built.

---

## Authoring Rules

1. **Overview must be specific.** "What" is one paragraph. "Why" links to or summarizes the relevant architecture doc and UX/PMF doc sections.
2. **Out of scope is mandatory.** List at least two explicit exclusions. If you cannot name two things out of scope, the scope is not well-defined.
3. **Each session is self-contained.** A developer reading only that session block should know exactly what to build, what files to touch, and how to verify it.
4. **Deliverables & Tests are combined.** Each deliverable has an associated test or verification method. No deliverable without a test.
5. **Session size estimates are required.** S = <200 LOC, M = 200–500 LOC, L = 500–1000 LOC. Sessions estimated >L must be split.
6. **Session Order & Dependencies is generated by scope-limiter.** Do not pre-fill this section.
7. **Open Questions block blockers.** Anything listed here means the plan cannot be executed until resolved. Do not add speculative questions.

---

## Template

```markdown
## Implementation Guide: [Feature / Change Name]
_Last updated: [date] | Status: Draft / Approved_

### Overview
- **What:** (one paragraph describing what will be built or changed)
- **Why:** (link to or summary of relevant architecture doc + UX/PMF doc sections; why this matters)
- **Out of scope:** (explicit list of at least two things NOT being done here)

### Sessions

#### Session N: [Name]
**Goal:** (one sentence — what is achieved by the end of this session)
**Deliverables & Tests:**
  - [ ] Deliverable: [what is produced]
    Test: [how it is verified]
  - [ ] Deliverable: [what is produced]
    Test: [how it is verified]
**Files affected:** (list of files to be created or modified)
**Docs to update:** (list of docs that must be updated after this session)
**Estimated size:** S (<200 LOC) | M (200–500) | L (500–1000)

[...repeat for each session...]

### Session Order & Dependencies
_(Generated by scope-limiter agent — not pre-filled)_
- Session order: ...
- Blocking dependencies: ...
- Sessions that can run in parallel: ...

### Open Questions
_(Unresolved items that would change the plan — must be resolved before execution)_
- [ ] ...
```

---

## Quality Checks (orchestrator applies before presenting to user)

- [ ] Every session has a one-sentence goal
- [ ] Every deliverable has a test
- [ ] No session exceeds L size estimate
- [ ] "Out of scope" has at least two entries
- [ ] "Session Order & Dependencies" is filled by scope-limiter (not empty, not pre-filled manually)
- [ ] Naming is consistent across all sessions (same names for same things)
- [ ] No session references a file or module not introduced earlier in the guide or already existing
- [ ] Open Questions block only true blockers — nothing speculative
